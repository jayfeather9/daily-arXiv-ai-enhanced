### [1] [AI/ML Model Cards in Edge AI Cyberinfrastructure: towards Agentic AI](https://arxiv.org/abs/2511.21661)
*Beth Plale,Neelesh Karthikeyan,Isuru Gamage,Joe Stubbs,Sachith Withana*

Main category: cs.DC

TL;DR: 关联领域：DSL（否）、图处理（否）、MLIR（否）、编译器（否）、HLS（否）。
TLDR: 传统的 AI/ML 模型卡是静态的，无法反映模型在其生命周期中的实际使用情况。本文介绍了嵌入在 ICICLE AI Institute 生态系统中的动态对象——Patra Model Cards。研究重点评估了采用 Model Context Protocol (MCP) 作为 Patra 模型卡服务器接口的利弊：**定量分析**显示 MCP 相比 REST 接口存在开销；但**定性分析**表明，MCP 能实现“活动会话”，这对于动态模型卡获取和使用模型上下文信息至关重要，证明了 MCP 在此特定场景中的价值和适用性。

Motivation: 传统的 AI/ML 模型卡通常只在训练时进行一次基准评估，无法捕捉模型在整个生命周期中实际如何以及在哪里被使用。这导致模型卡无法反映模型的真实使用上下文和动态变化。
本文的动机是希望通过将模型卡视为“动态对象”，并在 ICICLE AI Institute 软件生态系统中嵌入 Patra Model Cards 来解决这一问题。具体来说，研究旨在评估 Model Context Protocol (MCP) 是否能作为一种有效的接口协议，支持模型卡动态地收集和反映其在使用过程中的上下文信息（即启用活动会话），从而增强模型卡的实用性和实时性。

Method: 研究方法包括：1. **构建 Patra Model Cards**，并将其嵌入到 ICICLE AI Institute 软件生态系统中，以作为“动态对象”进行研究。2. **评估 Model Context Protocol (MCP)** 作为 Patra Model Card 服务器接口的适用性。3. **进行定量评估**，比较 MCP 接口与传统的 REST 接口在开销上的差异（即性能损耗）。4. **进行定性评估**，探讨 MCP 所实现的“活动会话”功能在动态模型卡背景下的适用性和价值（即功能契合度）。

Result: 1. **Patra 模型卡被确立为动态对象**，能够集成到软件生态系统（ICICLE AI Institute）中。2. **定量结果**：将 Model Context Protocol (MCP) 作为 Patra Model Card 服务器的接口，相比传统的 REST 接口，会引入性能开销（overhead）。3. **定性结果**：MCP 能够实现“活动会话”（active sessions），这是一个关键特性，解决了动态模型卡对模型使用上下文（Context）信息的收集和反映需求。研究同时评估了 MCP 在动态模型卡背景下的适用性和功能契合度。

Conclusion: 本研究评估了将 Model Context Protocol (MCP) 作为 Patra Model Card 服务器接口的利弊。虽然定量分析显示 MCP 相比 REST 接口有额外的开销，但更重要的结论在于 MCP 能够实现“活动会话”（active sessions），从而支持动态模型卡的功能需求。这证明了在需要动态上下文信息的场景中，即使存在性能开销，采用 MCP 仍然具有定性上的价值和适用性。

Abstract: AI/ML model cards can contain a benchmarked evaluation of an AI/ML model against intended use but a one time assessment during model training does not get at how and where a model is actually used over its lifetime. Through Patra Model Cards embedded in the ICICLE AI Institute software ecosystem we study model cards as dynamic objects. The study reported here assesses the benefits and tradeoffs of adopting the Model Context Protocol (MCP) as an interface to the Patra Model Card server. Quantitative assessment shows the overhead of MCP as compared to a REST interface. The core question however is of active sessions enabled by MCP; this is a qualitative question of fit and use in the context of dynamic model cards that we address as well.
